# Agent Specification — GPT-4.1-nano HR Chatbot (CLI)

> **Project:** gpt-4.1-nano-responses-vectors-demo
> **Created:** 2026-03-04
> **Status:** Draft

---

## 1. Objective

Create a simple Python script that tests the **GPT-4.1-nano** model and the **Responses API** together with the **File Vector Store API** through the command line.

The model is deployed in a **Microsoft Foundry** project. As a use-case theme, the chatbot is designed to answer **Human Resources (HR)** related questions using grounded context from uploaded HR policy documents.

---

## 2. Technical Requirements

| Requirement | Detail |
|---|---|
| **Language** | Python 3.14.3 only |
| **Frameworks** | None — no web frameworks, no orchestration frameworks |
| **Cloud API** | Azure OpenAI / Microsoft Foundry Responses API |
| **Authentication** | Azure Managed Identity (`DefaultAzureCredential`) |
| **Code style** | As simple as possible; single-file preferred |
| **Documentation** | All code documented with inline comments in **English** |
| **Error handling** | Wrap every external resource call (API) and every IO interaction in `try/except` blocks |

### 2.1 Key APIs

| API | Purpose |
|---|---|
| **Responses API** | Send user prompts to the GPT-4.1-nano model and receive completions |
| **File Vector Store API** | Upload HR documents, create a vector store, and attach it as a file-search tool so the model can ground its answers on the uploaded content |

### 2.2 Authentication Flow

1. The script uses `azure-identity` with `DefaultAzureCredential`.
2. A token is obtained for the Azure AI / Foundry endpoint scope.
3. The token is passed as a `Bearer` token in the `Authorization` header of every API request.
4. No API keys are stored or used — Managed Identity only.

## 4. User Experience

The interaction follows a **four-step linear flow**:

```
Step 1 ─► Step 2 ─► Step 3 ─► Step 4
 Run       Intro     Prompt    Answer
```

### Step 1 — Run the script

```bash
python main.py
```

### Step 2 — Script explains its purpose

The script prints a welcome banner and a short explanation of what it does, e.g.:

```
===================================================
  HR Assistant — Powered by GPT-4.1-nano
===================================================

This tool answers Human Resources questions using
company HR policy documents as context.

Type your question below or type "exit" to quit.
```

### Step 3 — Prompt the user

```
You: <user types a question here>
```

### Step 4 — Return the answer

```
Assistant: <model response grounded on the uploaded HR documents>
```

Steps 3 and 4 repeat in a loop until the user types `exit` or `quit`.

## 5. Environment Variables

The script must read configuration from environment variables (no hard-coded secrets):

| Variable | Required | Description |
|---|---|---|
| `AZURE_FOUNDRY_ENDPOINT` | Yes | The Microsoft Foundry / Azure AI endpoint URL (e.g., `https://<project>.services.ai.azure.com`) |
| `AZURE_FOUNDRY_MODEL` | No | Model deployment name. Defaults to `gpt-4.1-nano` |

---

## 6. Functional Flow (main.py)

```
1. Read environment variables
2. Authenticate via DefaultAzureCredential (Managed Identity)
3. Upload each file from docs/ using the Files API
4. Create a Vector Store and attach the uploaded files
5. Poll until the Vector Store status is "completed"
6. Print the welcome banner (Step 2)
7. Loop:
   a. Prompt user for input (Step 3)
   b. If input is "exit"/"quit" → break
   c. Call the Responses API with:
      - model: gpt-4.1-nano
      - input: user question
      - tools: [file_search] linked to the vector store
   d. Parse and print the model's text response (Step 4)
8. Cleanup (optional): delete vector store and uploaded files
```

---

## 7. Error Handling Strategy

| Scenario | Handling |
|---|---|
| Missing environment variables | Print a clear error message and exit with code 1 |
| Authentication failure | Catch `ClientAuthenticationError`, print guidance, exit |
| File upload failure | Catch exception per file, log a warning, continue with remaining files |
| Vector Store creation/polling failure | Catch exception, print error, exit |
| Responses API call failure | Catch exception, print friendly error, allow user to retry |
| User input (IO) | Wrap `input()` in try/except for `KeyboardInterrupt` and `EOFError` |

---

## 8. Constraints & Non-Goals

- **No web server** — this is a CLI-only tool.
- **No database** — all context comes from the docs/ folder via the Vector Store API.
- **No streaming** — responses are returned as a single block (keep it simple).
- **No conversation memory** — each question is independent (stateless).
- **No third-party frameworks** — no LangChain, Semantic Kernel, or similar.
